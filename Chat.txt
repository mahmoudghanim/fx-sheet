4. TECHNICAL ARCHITECTURE
================================================================================

Frontend:
- Single HTML file with embedded CSS and JavaScript
- No external dependencies (pure vanilla JS)
- Fetches data from stocks-data.json via HTTP
- Client-side filtering, sorting, and pagination

Backend:
- Simple Python HTTP server (server.py)
- Serves static files from local directory
- Alternative: Flask app (flask_app.py) for production

Data Flow:
1. fetch_yahoo_data.py queries Yahoo Finance API
2. Data saved to stocks-data.json (140 KB)
3. server.py serves files on http://localhost:8001
4. index.html fetches stocks-data.json
5. JavaScript renders table with pagination
6. User filters/sorts client-side

Batch Processing:
- Fetches 50 stocks per request
- 10 batches total for 463 stocks
- ~5-10 minutes for complete update
- Progress indication per user preference

5. DATA SOURCE STRATEGY
================================================================================

PRIMARY: Yahoo Finance API (yfinance)
✓ Free, no authentication required
✓ Works with any Python version (3.7+)
✓ Real-time stock prices
✓ Industry and sector data
✓ Currency information
✓ 52-week high/low
✓ Volume data
✓ Change percentages
✓ 86.6% success rate (401/463 stocks)
✓ Batch processing with progress feedback

ALTERNATIVE: FXCM Forex Connect API
✗ Deprecated (ORA-499 error)
✗ Requires Python 3.7 specifically
✗ Needs Forex Connect SDK installation
✗ Authentication required
✓ Kept as fallback (--fxcm flag in update_stocks.py)

Symbol Mapping:
- Converts FXCM format to Yahoo Finance format
- Example: ABBV.us → ABBV, ADS.de → ADS.DE
- Supports 40+ exchanges worldwide
- Automatic suffix mapping (US, .L, .DE, .PA, etc.)

Failed Stocks (62):
- Primarily Hong Kong stocks (different Yahoo Finance format)
- Some Canadian stocks with special suffixes
- Stocks with multiple listings

OPTION 3: VPS/Cloud ($5-20/month)

1. Upload all files
2. pip install flask gunicorn
3. gunicorn --bind 0.0.0.0:8001 flask_app:app

5. Set up cron for daily updates
Cost: $5-20/month
Auto-update: Cron job (0 6 * * * python fetch_yahoo_data.py)

OPTION 4: Docker (Any Server)
Files: Dockerfile, docker-compose.yml
Build: docker build -t fx-sheet .
Run: docker run -d -p 8001:8001 fx-sheet
Cost: Varies by host

9. TROUBLESHOOTING
================================================================================

Problem: "Python is not installed"
Solution: Install Python from https://www.python.org/downloads/
         Check "Add Python to PATH" during installation

Problem: Port 8001 already in use
Solution: Close other dashboard windows
         Or change PORT in server.py
         Server now has better error messages

Problem: Dashboard showing old data
Solution: Run update_data.bat
         Refresh browser with Ctrl+Shift+R
         Clear browser cache

Problem: Industry showing N/A
Solution: Run fetch_yahoo_data.py to fetch with industry data
         Data must include industry field

Problem: Currency showing N/A
Solution: Run fetch_yahoo_data.py to fetch with currency data
         Data must include currency field

Problem: GME showing wrong price ($485)
Solution: Was browser cache issue
         Hard refresh with Ctrl+Shift+R
         Data was correct in JSON ($24.52)

Problem: FXCM ORA-499 error
Solution: FXCM API is deprecated
         Use Yahoo Finance instead (default)
         FXCM kept as alternative (--fxcm flag)

Problem: Hong Kong stocks failing
Solution: Different Yahoo Finance symbol format
         62 stocks fail out of 463 (86.6% success)
         This is expected, not a bug

Problem: Slow page load
Solution: Pagination implemented (50 per page default)
         10x faster than loading all 463 at once
         Change page size if needed

Problem: Filters not working
Solution: Click "Apply Filters" button
         Check filter values are valid numbers
         Clear filters and try again

KEY DECISIONS MADE
================================================================================

1. Data Source: Yahoo Finance over FXCM
   Reason: FXCM API deprecated, Yahoo Finance free and reliable

2. Pagination over Virtual Scrolling
   Reason: Simpler implementation, sufficient for 463 stocks

3. Static File Architecture
   Reason: Easy deployment, works on any host, no database needed

4. Client-side Filtering
   Reason: Fast, no server load, instant feedback

5. Compact UI Design
   Reason: Show more data on screen, user requested

6. Yahoo Finance Links
   Reason: Quick access to detailed stock information

7. Industry & Currency Filters
   Reason: Better stock categorization and analysis

8. Batch Processing (50 per request)
   Reason: Avoid rate limits, show progress, reliable
